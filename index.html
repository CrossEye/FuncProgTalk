What is Functional Programming?

No Single Definition

There is no clear, widely-accepted definition of Functional Programming. It is a collection of related featues which cohere well into a very useful style of programming, including:

  • First-class functions
  • Lambdas/Anonymous Functions with closures
  • Compact, even terse, functions
  • Mostly stateless processing, side-effect-free function calls
  • Performant recursion through tail call optimization
  • Pattern matching (Haskell, Erlang)
  • Lazy Evaluation (Miranda, Haskell)
  • Homoiconicity (mostly LISP-like languages?)

Not all are available in Javascript

The first three are freely available; the fourth we can mostly achieve with some discipline; the others are beyond the abilities of the language. We won't discuss them much.
